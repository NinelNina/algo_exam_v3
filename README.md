# Итоговое задание по дисциплине "Алгоритмы и структуры данных"
# Вариант 3. Нечёткий поиск, выявление дублирующихся значений

Крупный маркетплейс столкнулся с проблемой дублирования товарных предложений. Разные продавцы добавляют один и тот же товар, но в его описании могут быть опечатки, различные сокращения, синонимы, изменённый порядок слов или дополнительные характеристики. Это ухудшает пользовательский опыт и мешает системе корректно ранжировать предложения. Необходимо разработать алгоритм, который для каждого нового товара будет находить в каталоге потенциальные дубликаты.

## Проблема
Разные продавцы могут добавлять один и тот же товар с названиями, оформленными по-разному:
- `Смартфон Xiaomi Redmi Note 12 Pro 8/256GB синий`
- `Xiaomi Redmi Note 12 Pro 8/256 Синий`

## Решение
Простое сравнение строк не справляется с такими вариациями, поэтому было применено решение, основанное на:
1. **Текстовой предобработке, состоящей из**:
   - Приведения к нижнему регистру
   - Унификации цветов (`синий` -> `blue`), объёма памяти (`256ГБ` -> `256`), единиц измерения
   - Удаления лишних символов
2. **Гибридной мере схожести на базе двух алгоритмов**:
   - Коэффициента Жаккара по словам (устойчив к перестановкам слов)
   - Нормализованного расстояния Левенштейна (ловит мелкие различия)
Программа позволяет настроить порог схожести и включить/отключить предобработку.

## Формат данных
- **Вход**:  
  - `catalog.txt` - существующий каталог товаров, где файлы представлены в формате `ID <название товара>`
  - `new_items.txt` - новые товары (в том же формате)
- **Выход**:  
  - `duplicates.json` - для каждого нового товара список потенциальных дубликатов с коэффициентом схожести

## Как запустить проект
1. Поместите файлы `catalog.txt` и `new_items.txt` в корневую папку проекта.
2. Настройте параметры в конфигурации (порог схожести и предобработку, по умолчанию `SIMILARITY_THRESHOLD = 0.8`, `USE_PREPROCESSING = True`.
3. Запустите скрипт через терминал:
   ```bash
   python main.py
